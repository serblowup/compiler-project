# Семантический анализ

## Обзор

Семантический анализатор выполняет проверку корректности программы на уровне типов и областей видимости после успешного синтаксического разбора. Он строит таблицу символов, проверяет типы выражений и декорирует AST аннотациями типов.

## Таблица символов

Таблица символов реализована с поддержкой вложенных областей видимости.

### Типы областей видимости

| Тип | Описание |
|-----|----------|
| `GLOBAL` | Глобальная область (уровень программы) |
| `FUNCTION` | Область функции (параметры и локальные переменные) |
| `BLOCK` | Блочная область (внутри `{ ... }`) |
| `STRUCT` | Область структуры (имена полей) |
| `IF_THEN` | Ветка `then` оператора `if` |
| `IF_ELSE` | Ветка `else` оператора `if` |
| `FOR_LOOP` | Заголовок цикла `for` |
| `FOR_BODY` | Тело цикла `for` |
| `WHILE_BODY` | Тело цикла `while` |

### Операции с таблицей символов

```cpp
// Создание новой области видимости
symbolTable.enterScope(ScopeType::FUNCTION, "functionName");

// Выход из текущей области
symbolTable.exitScope();

// Вставка символа в текущую область
symbolTable.insert("variableName", symbolInfo);

// Поиск символа (от текущей до глобальной)
SymbolInfo* sym = symbolTable.lookup("name");

// Поиск только в текущей области
SymbolInfo* sym = symbolTable.lookupCurrent("name");
```

## Система типов

### Базовые типы

| Тип | Размер | Выравнивание |
|-----|--------|--------------|
| `int` | 4 байта | 4 |
| `float` | 4 байта | 4 |
| `bool` | 1 байт | 1 |
| `void` | 0 байт | - |
| `string` | 8 байт | 8 |

### Составные типы

- **Структуры** - пользовательские типы с именованными полями
- **Функции** - тип с параметрами и возвращаемым значением

### Правила совместимости типов

1. **Точное совпадение** - типы идентичны
2. **Неявное преобразование**:
   - `int` → `float` (расширение)
   - `int` → `bool` (ненулевое значение → true)
   - `float` → `bool` (ненулевое значение → true)
3. **Запрещенные преобразования**:
   - `float` → `int` (требуется явное приведение)
   - `void` → любой тип

## Проверка выражений

### Бинарные операторы

| Операторы | Типы операндов | Тип результата |
|-----------|----------------|----------------|
| `+`, `-`, `*`, `/`, `%` | `int` + `int` | `int` |
| `+`, `-`, `*`, `/`, `%` | `float` + `float` | `float` |
| `+`, `-`, `*`, `/`, `%` | `int` + `float` | `float` |
| `==`, `!=`, `<`, `>`, `<=`, `>=` | `T` + `T` (числовые) | `bool` |
| `&&`, `||` | `bool` + `bool` | `bool` |

### Унарные операторы

| Оператор | Тип операнда | Тип результата |
|----------|--------------|----------------|
| `-` | `int` | `int` |
| `-` | `float` | `float` |
| `!` | `bool` | `bool` |
| `++`, `--` | `int`, `float` | Тот же тип |

### Присваивание

- Тип левой части должен быть совместим с типом правой части
- Левая часть должна быть изменяемой (не константа)
- Составные операторы (`+=`, `-=`, и т.д.) требуют арифметического типа

## Проверка функций

### Объявление функции

```c
fn имя(тип1 param1, тип2 param2) -> возвращаемый_тип {
    // тело функции
}
```

Проверки:
- Имя функции уникально в глобальной области
- Тип возвращаемого значения корректен
- Имена параметров уникальны в пределах функции

### Вызов функции

```c
результат = имя(аргумент1, аргумент2);
```

Проверки:
- Количество аргументов соответствует количеству параметров
- Типы аргументов совместимы с типами параметров
- Возвращаемое значение используется корректно

### Оператор return

- В `void` функции запрещен `return` с значением
- В не-`void` функции должен быть `return` с совместимым типом

## Управляющие конструкции

### if / else

```c
if (условие) {
    // then ветка
} else {
    // else ветка
}
```

- Условие должно иметь тип `bool` или быть преобразуемым к `bool`
- Создаются отдельные области видимости для then и else веток

### while

```c
while (условие) {
    // тело цикла
}
```

- Условие должно иметь тип `bool` или быть преобразуемым к `bool`
- Тело цикла выполняется в отдельной области видимости

### for

```c
for (инициализация; условие; шаг) {
    // тело цикла
}
```

- Инициализация может быть объявлением переменной или выражением
- Условие должно иметь тип `bool` или быть преобразуемым к `bool`
- Шаг выполняется после каждой итерации

## Структуры

### Объявление структуры

```c
struct Имя {
    тип1 поле1;
    тип2 поле2;
};
```

Проверки:
- Имя структуры уникально в глобальной области
- Имена полей уникальны в пределах структуры
- Типы полей объявлены

### Доступ к полям

```c
переменная_структуры.поле
```

- Левая часть должна иметь тип структуры
- Поле должно существовать в структуре

## Обработка ошибок

### Категории ошибок

| Код ошибки | Описание |
|------------|----------|
| `UNDECLARED_IDENTIFIER` | Идентификатор не объявлен |
| `DUPLICATE_DECLARATION` | Повторное объявление в одной области |
| `TYPE_MISMATCH` | Несоответствие типов |
| `ARGUMENT_COUNT_MISMATCH` | Неправильное количество аргументов |
| `ARGUMENT_TYPE_MISMATCH` | Несоответствие типов аргументов |
| `INVALID_RETURN_TYPE` | Неверный тип возвращаемого значения |
| `INVALID_CONDITION_TYPE` | Условие не является bool |
| `UNINITIALIZED_VARIABLE` | Использование неинициализированной переменной |
| `VOID_VARIABLE` | Переменная типа void |
| `INVALID_OPERATION` | Недопустимая операция для данного типа |

### Формат сообщения об ошибке

```
semantic error: <тип_ошибки> - <описание>
  --> <файл>:<строка>:<колонка>
   |
<строка> | <код_строки>
   |      <указатель>
   |
   = ожидалось: <тип>
   = получено: <тип>
   = подсказка: <предложение_по_исправлению>
```

## Размещение в памяти (Memory Layout)

Для каждой функции вычисляется:
- Смещение каждого параметра и локальной переменной в стеке
- Размер каждой переменной в байтах
- Общий размер стека функции (с выравниванием до 16 байт)

## Отчет о валидации

При запуске с флагом `--verbose` выводится подробный отчет:

1. **Сводка по ошибкам** - количество ошибок
2. **Символы по областям видимости** - таблица символов
3. **Иерархия типов** - все типы с размерами и выравниванием
4. **Статистика типов** - распределение типов в выражениях
5. **Размещение в памяти** - стеки функций со смещениями переменных

## Примеры

### Корректная программа

```c
fn factorial(int n) -> int {
    int result = 1;
    int i = 1;
    
    while (i <= n) {
        result = result * i;
        i = i + 1;
    }
    
    return result;
}

fn main() -> void {
    int x = factorial(5);
    return;
}
```

### Ошибка: несоответствие типов

```c
fn main() -> void {
    int x;
    x = "hello";  // Error: cannot assign string to int
}
```

Вывод:
```
semantic error: несоответствие типов - в присваивании
  --> test.src:3:9
   |
3 |     x = "hello";
   |         ^^^^^^^
   |
   = ожидалось: int
   = получено: string
   = подсказка: приведите значение к ожидаемому типу
```

### Ошибка: необъявленная переменная

```c
fn main() -> void {
    y = 10;  // Error: 'y' undeclared
}
```

Вывод:
```
semantic error: необъявленный идентификатор - идентификатор 'y' не объявлен
  --> test.src:2:5
   |
2 |     y = 10;
   |     ^
   |
   = идентификатор: 'y'
   = подсказка: проверьте, объявлена ли переменная 'y' в текущей области видимости
```
